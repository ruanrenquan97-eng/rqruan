# 前端项目全栈部署指南 (Git + GitHub + Vercel)

本指南详细记录了从零开始，将一个本地项目部署到云端免费平台（Vercel），并实现全自动化更新上线的完整流程。

---

## 核心架构原理

整个部署流水线遵循以下**单向自动化链路**：

1. **本地电脑 (Workspace)**：编写代码、测试代码。
2. **GitHub (代码仓库)**：云端代码托管中心。
3. **Vercel (部署平台)**：监听 GitHub 变化，自动拉取代码并提供全球 CDN 访问网址。

它们能够产生关联是因为在各个环节配置了“监听”和“目标地址”。

---

## 第一阶段：本地初始化与关联 GitHub (仅需配置一次)

假设你在电脑上新建了一个名为 `my-new-project` 的网站项目文件夹。

### 1. 初始化本地仓库
打开终端，**进入项目根目录**，执行：
```bash
git init
```
*这会在文件夹中创建一个隐藏的 `.git` 文件夹，让当前目录拥有了被 Git 追踪代码变动的能力。*

### 2. 第一次提交代码
```bash
# 暂存当前文件夹下的所有代码
git add . 

# 提交并打上初始标签
git commit -m "Initial commit"
```

### 3. 在 GitHub 上创建云端仓库
- 登录 GitHub，点击右上角 `+` 号选择 **New repository**。
- 输入仓库名称（如 `my-new-project`），选择 Public 或 Private。
- **⚠️ 关键步骤：不要勾选 Add a README 或 .gitignore**，保持仓库完全为空。
- 点击 **Create repository**。

### 4. 绑定本地代码到 GitHub
复制 GitHub 生成的仓库链接，然后在本地终端运行以下命令，将刚才提交的代码推送上去：
```bash
# 添加云端仓库的关联记录（将那个网址命名为 'origin' 大哥）
git remote add origin https://github.com/你的名字/my-new-project.git

# 将本地代码推送到 'main' 分支
git push -u origin main
```
*从此你的电脑就“认准”了这个 GitHub 仓库。*

---

## 第二阶段：部署至 Vercel (仅需配置一次)

为了让全世界的人都能通过网址访问你的网站，我们需要将 GitHub 的代码导入到部署平台 Vercel。

### 1. 登录并导入项目
- 打开 [Vercel 官网](https://vercel.com/) 并使用 GitHub 账号授权登录。
- 在控制台右上角点击 **Add New... -> Project**。
- Vercel 会读取你的 GitHub 仓库列表，找到 `my-new-project`，点击右侧的 **Import**。

### 2. 构建与部署
- 在配置页面通常**不需要修改任何东西**（Vercel 会自动识别你用的是 React/Vite/Vue 等框架）。
- 直接点击蓝色的 **Deploy** 按钮。
- 等待 1~2 分钟撒花庆祝，Vercel 会自动为你生成专属的访问链接（如 `my-new-project.vercel.app`）。

> **💡 配置 Vercel 的 Webhook 原理：**  
> 当你点击 Import 授权时，Vercel 就在你的 GitHub 对应的仓库里种下了一个 Webhook（钩子）。它就像一个门铃，未来 GitHub 只要有任何更新，就会立刻按响 Vercel 的门铃通知它拉取最新代码。

---

## 第三阶段：日常全自动更新 (永久循环)

未来每一天，当你写完新功能、修改了 Bug，或者只是改了一个标题。你只需要：

1. 打开项目的本地终端。
2. 敲下这三行“咒语”：
```bash
git add .
git commit -m "写上你这次改了什么"
git push
```
3. 收工。

**背后发生的事：**
- 修改被推送到 GitHub。
- GitHub 立即触发 Vercel 的门铃（Webhook）。
- Vercel 自动开启一台机器，拉取最新代码，用 `npm` 安装依赖，并打包（Build）编译。
- 编译成功后，Vercel 自动将新网页平滑替换掉旧网页。

几分钟后，刷新浏览器，你的网址里展示的就是最新版本的代码了。真正实现了**本地写码，云端自动构建上线**。
